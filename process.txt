1. Send data through sender class. Sender thread will catch, and pack it to send over to server through simulated network.

2. Receiving thread catches packet one by one, placing it in received buffer, awaiting for current sequence and total packets to match up.
    2.5. NES_ACK ONLY: In case of missing packet, current sequence will not move forward until packet is received. 
    2.5. SEL_REP_ACK ONLY: In case of missing packet, current sequence will finish, but any missing packet from the buffer will requested, one by one, until buffer is filled.

    2.6 To all: In case a packet never arrives, there are timers, in which when hit a certain limit, will send to sender a request to resend packet.

3. Receiver will finalize data, reforming data by packet order, and will place available data into the awaiting collection buffer.

4. Call receive function through receiver class to collect latest data from the awaiting collection buffer.

Note: This is how I think it goes and how I based of the program to work by. This can easily be adapted to another form and can even be used outside.
